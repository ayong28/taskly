## What happened this session

Two pieces of work, each done test-first (red → green):

1. **Drag-and-drop bug fix** (commits `a960423`, `2df6f08`, and DragOverlay follow-up `8e74ec8`/`f53a737`): dropping a card into an *empty* list was a no-op because empty lists had no droppable target. Fixed in `components/Board/BoardCanvas.tsx` via a `useDroppable` wrapper per list (`list-{id}` id) and updated `handleDragEnd` to treat that as "append to end." Also added a floating "held card" visual in `DragOverlay` (was previously empty).

2. **Step 7 — Labels** (commit `eb9915d`): global (not board-scoped) labels, preset color palette, created/renamed/deleted inline from a card's label popover, assignable/removable per card, shown as dot swatches on `CardTile` and pills in `CardModal`.
   - New files: `lib/labels.ts` (palette), `lib/actions/labels.ts` (server actions), `components/Label/LabelChip.tsx`, `components/Label/LabelPicker.tsx`
   - Modified: `CardModal`, `CardTile`, `SortableCardTile`, `BoardCanvas`, `app/board/[id]/page.tsx` (loads global labels + per-card assignment map)
   - **A real bug was caught and fixed before it was ever committed**: base-ui's `Menu` primitive intercepts every character keydown for typeahead navigation (jump-select menu items by letter), which swallowed keystrokes meant for the plain `<input>` fields nested inside the label popover (the "new label name" and "rename label" inputs). Fixed by adding `onKeyDown={(e) => e.stopPropagation()}` on both inputs in `components/Label/LabelPicker.tsx`.
   - **Testing lesson learned, now applied**: Playwright's `.fill()` sets input values directly and bypasses real keydown events, so it did *not* catch the above bug. The E2E test (`e2e/labels.spec.ts`) uses `.pressSequentially()` instead for those two specific fields, since that dispatches real per-key events. Default to `.fill()` for ordinary form fields, but reach for `.pressSequentially()` when a field is nested inside any listbox/menu/combobox-like widget that might have its own keyboard handling.

## Testing approach established this session (apply going forward)

For any new feature, split tests as:
- **Jest** for component/logic wiring — mock out third-party primitives that don't work well in jsdom (`@dnd-kit/core`, `@dnd-kit/sortable`, `@/components/ui/dropdown-menu`) and mock server actions (`@/lib/actions/*`) and `next/navigation`'s `useRouter`. See `components/Board/BoardCanvas.test.tsx` and `components/Label/LabelPicker.test.tsx` for the established mocking patterns.
- **Playwright** for real end-to-end flows and persistence-after-reload checks.
- Always write the test first, run it to confirm red, then implement.

## Environment notes

- `npx playwright test` starts its own dev server on port 3001 (see `playwright.config.ts`). If the user has their own `npm run dev` running (any port), Next's lock file blocks Playwright's server from starting — ask the user to stop their manual dev server before running the E2E suite, don't kill it yourself.
- Test DB lives at `data/test.db` (WAL mode) — safe to delete (`data/test.db*`) between runs if state gets weird; the real app DB is `data/project-manager.db`, never touch that.
- `workers: 1` is set intentionally in `playwright.config.ts` to avoid parallel writes to the shared SQLite test DB.
